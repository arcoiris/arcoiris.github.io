Oh, ActiveRecord Is Not Exclusive to Rails"

Aug 2nd, 2014

Recently on a tealeaf academy quiz, I didn’t know how to sensibly: “At a high level, describe the ActiveRecord pattern.”

Active Record is a way of accessing data in a database. Active Record is a pattern that can be implemented outside of the Rails framework. I didn’t understand that. example: PHP active record

Active Record maps:

  1. Classes to tables:

     class post < ActiveRecord::Base 
 end

  2. Object to rows of data within that table #

  3. Getters/setters to columns in that table post = Post.first post.title = “Ruby Documentation” # setter post.title # getter
